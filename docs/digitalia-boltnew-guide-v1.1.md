## 1. プロジェクト概要

### アプリの目的
プログラミング初心者が、ファンタジー世界「デジタリア」を冒険しながら、520の専門用語を**ストーリー体験型学習**で楽しく学べるWebアプリ

### コンセプト：3つの学習モード
| モード | 説明 | 特徴 |
|--------|------|------|
| **ストーリーモード** | テレプロンプター形式でストーリーを体験 | メイン体験、没入感重視 |
| **フラッシュカードモード** | 従来のカード学習 | 復習・反復学習向け |
| **検索・辞書モード** | 用語を直接検索・参照 | リファレンスとして使用 |

### コア体験フロー
```
ストーリーを読む/聴く
    ↓
用語がハイライトで登場
    ↓
タップして用語カードを見る
    ↓
「理解した！」を押す
    ↓
🦋 蝶が誕生 → マイフォルダへ格納
    ↓
ストーリー再開
```

### 用語カード構成

**【表面】**
- カテゴリアイコン
- 難易度（★〜★★★）
- 用語名（英語）
- 日本語読み
- 一言説明

**【裏面】**
1. 📖 詳細定義
2. ⚙️ どういう仕組み？
3. 💡 例えるなら
4. ⚠️ 注意点
5. 🔗 関連用語（タップでジャンプ）

## 2. 事前準備

### 2.1 Bolt.newとは？

Bolt.newは、自然言語（日本語OK）でWebアプリを作れるAI開発ツールです。
コードを書かずに「こんなアプリを作って」と指示するだけで、動くアプリが生成されます。

### 2.2 アカウント準備

1. **Bolt.new にアクセス**
   - URL: https://bolt.new
   
2. **アカウント作成**
   - GitHubまたはGoogleアカウントでログイン
   
3. **プラン確認**
   - 無料プランでも基本機能は使用可能

## 4. Phase 2：データ設計

### 4.1 用語データの型定義

**Bolt.newへの指示：**

```
用語データの型定義を作成してください。

【Term（用語）の構造】
interface Term {
  id: string;                    // 一意のID（例: "variable"）
  term: string;                  // 英語の用語名
  termJa: string;                // 日本語読み
  category: string;              // カテゴリID
  chapter: number;               // 章番号（1-17、17はエピローグ）
  order: number;                 // 章内の順番
  difficulty: 1 | 2 | 3;         // 難易度
  oneLiner: string;              // 一言説明
  definition: string;            // 詳細な定義
  mechanism: string;             // 仕組み・実装方法
  analogy: string;               // 例え話
  pitfall: string;               // 注意点
  relatedTerms: string[];        // 関連用語のID配列
  storyContext: string;          // ストーリー内での登場文脈
}

【Chapter（章）の構造】
interface Chapter {
  id: number;
  title: string;
  subtitle: string;
  category: string;
  categoryName: string;
  categoryIcon: string;          // Lucideアイコン名
  description: string;
  termCount: number;
  guide: {
    name: string;
    role: string;
    personality: string;
    speechStyle: string;
  };
}

types/term.ts と types/chapter.ts に分けて作成してください。
```

- 上部：キャラクター表示（小さめ）

## 12. トラブルシューティング

### よくある問題と解決策

| 問題 | 原因 | 解決策 |
|------|------|--------|
| 画面が真っ白 | JavaScriptエラー | コンソールを確認し、エラーを報告 |
| データが保存されない | LocalStorage制限 | データ量を確認、古いデータを削除 |
| アニメーションがカクカク | パフォーマンス不足 | will-changeプロパティを追加 |
| スマホで表示崩れ | CSS未対応 | メディアクエリを追加依頼 |
| 音声が再生されない | ブラウザ非対応 | ユーザーに通知、フォールバック表示 |
| 蝶のアニメーションが重い | 要素数過多 | 表示数を制限、仮想化 |
| キャラクター画像が表示されない | パス間違い | 画像パスを確認、public配下か確認 |

### Bolt.newが期待通りに動かない時

1. **指示を分割する**
   - 大きな機能は小さなステップに分けて依頼
   - 例：「ストーリーモード全体」→「テレプロンプター」→「用語ハイライト」→「音声読み上げ」

2. **具体例を示す**
   - 「こんな感じで」と参考画像やコードを添付
   - ASCIIアートでレイアウトを図示

3. **現状を正確に伝える**
   - 何が動いていて、何が動いていないか明確に
   - スクリーンショットを添付

4. **リセットも選択肢**
   - 行き詰まったら、その部分だけ作り直しを依頼
