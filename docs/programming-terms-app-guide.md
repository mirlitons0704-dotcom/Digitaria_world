## 1. プロジェクト概要

### アプリの目的
プログラミング初心者が、ストーリーを追いながら500の専門用語を楽しく学習できるWebアプリ

### コンセプト：3つの要素を融合
| 要素 | 内容 |
|------|------|
| **冒険型** | コンピュータの世界を旅する物語 |
| **成長型** | 主人公（＝学習者）がアプリ開発者として成長 |
| **解説型** | 案内役キャラクターが丁寧にガイド |

### カード構成

**【表面】**
- カテゴリアイコン
- 難易度（★〜★★★）
- 用語名
- 一言で言うと（5〜10語の本質）

**【裏面】**
1. 📖 詳細定義
2. ⚙️ どういう仕組み？
3. 💡 例えるなら
4. ⚠️ 初心者あるある
5. 🔗 関連用語（タップでジャンプ）

## 2. 事前準備

### 2.1 Bolt.newとは？

Bolt.newは、自然言語（日本語OK）でWebアプリを作れるAI開発ツールです。
コードを書かずに「こんなアプリを作って」と指示するだけで、動くアプリが生成されます。

### 2.2 アカウント準備

1. **Bolt.new にアクセス**
   - URL: https://bolt.new
   
2. **アカウント作成**
   - GitHubまたはGoogleアカウントでログイン
   
3. **プラン確認**
   - 無料プランでも基本機能は使用可能
   - 大規模プロジェクトはProプラン推奨

### 2.3 必要な素材の準備

制作開始前に以下を用意しておくとスムーズです：

```
📁 準備フォルダ
├── 📄 500語リスト.csv（用語データ）
├── 📄 ストーリー原稿.md（章ごとの物語）
├── 🖼️ キャラクター画像（案内役など）
└── 🎨 デザイン参考画像（UIイメージ）
```

## 3. Phase 1：基盤構築

### 3.1 プロジェクト作成

**Bolt.newへの最初の指示：**

```
プログラミング用語を学習するWebアプリを作成してください。

【技術スタック】
- React + TypeScript
- Tailwind CSS
- Framer Motion（アニメーション用）
- Lucide React（アイコン用）

【基本構成】
- ホーム画面
- 章選択画面
- ストーリー閲覧画面
- 用語カード画面
- 進捗確認画面

【デザイン方針】
- ダークモード基調（目に優しい学習環境）
- 温かみのある配色（学習への親しみやすさ）
- 適度なアニメーション（楽しさの演出）

まずはルーティングとナビゲーションの骨組みだけ作成してください。
```

### 3.2 フォルダ構造の確認

生成後、以下の構造になっているか確認：

```
src/
├── components/     # 再利用可能なUI部品
│   ├── Card/
│   ├── Navigation/
│   └── Progress/
├── pages/          # 各画面
│   ├── Home.tsx

│   ├── Story.tsx
│   └── TermCard.tsx
```

各画面間を移動できることをテスト：
- ホーム → 章選択 → ストーリー → カード → ホーム

## 4. Phase 2：データ設計

### 4.1 用語データの型定義

**Bolt.newへの指示：**

```
用語データの型定義を作成してください。

【Term（用語）の構造】
- id: 一意の識別子（例: "binary"）
- term: 用語名（例: "Binary"）
- termJa: 日本語名（例: "2進数"）
- category: カテゴリ（16種類のいずれか）
- difficulty: 難易度（1〜3）
- oneLiner: 一言説明（10語以内）
- definition: 詳細定義
- mechanism: どういう仕組み？
- analogy: 例えるなら
- pitfall: 初心者あるある
- relatedTerms: 関連用語のIDリスト

【Category（カテゴリ）は以下の16種類】
1. basics（基礎概念）
2. fileTypes（ファイル形式）
3. dataTypes（データ型・構造）
4. programming（プログラミング基礎）
5. htmlCss（HTML & CSS）
6. urlHttp（URL・HTTP）
7. javascript（JavaScript & TypeScript）
8. react（React）
9. uiComponents（UIコンポーネント）
10. uiux（UI/UX デザイン）
11. cssFrameworks（CSSフレームワーク）
12. devTools（開発ツール）
13. environment（環境・パッケージ管理）
14. git（バージョン管理）
15. backend（バックエンド）
16. api（API）
```

### 4.2 ストーリーデータの型定義

**Bolt.newへの指示：**

```
ストーリーデータの型定義を追加してください。

【Chapter（章）の構造】
- id: 章番号（1〜16）
- title: 章タイトル
- subtitle: サブタイトル
- description: 章の概要
- category: 対応するカテゴリ
- termIds: この章で登場する用語IDのリスト
- scenes: シーンの配列

【Scene（シーン）の構造】
- id: シーンID
- type: "narrative"（物語）| "termIntro"（用語登場）| "quiz"（クイズ）
- content: シーンの内容テキスト
- termId: （用語登場の場合）紹介する用語のID
- choices: （クイズの場合）選択肢の配列
```

## 5. Phase 3：コア機能開発

### 5.1 用語カードコンポーネント

**Bolt.newへの指示：**

```
フリップ式の用語カードコンポーネントを作成してください。

【表面の要素】

- 右上：難易度（★で表示）
- 中央上部：用語名（大きく）
- 中央下部：日本語名（やや小さく）
- 下部：一言説明

【裏面の要素】
- 📖 詳細定義（最大3行）
- ⚙️ どういう仕組み？
- 💡 例えるなら
- ⚠️ 初心者あるある
- 🔗 関連用語（クリック可能なチップ形式）

【アニメーション】
- カードをクリックで180度回転してフリップ
- Framer Motionを使用
- 回転中に裏面の内容が自然に見えるよう工夫
```

### 5.2 進捗管理システム

**Bolt.newへの指示：**

```
学習進捗を管理するシステムを実装してください。

【保存する情報】
- 各用語の習得状態（未学習/学習中/習得済み）
- 各章の進行状況（未開始/進行中/完了）
- クイズの正答率
- 最終学習日時

【機能】
- LocalStorageに保存（ブラウザを閉じても保持）
- 進捗率をパーセンテージで計算
- 章ごとの習得率を表示

【UIへの反映】
- 章選択画面で進捗バー表示
- ホーム画面で全体の習得数/500を表示
```

### 5.3 関連用語ジャンプ機能

**Bolt.newへの指示：**

```
関連用語をタップすると該当カードへジャンプする機能を実装してください。

【動作】
1. カード裏面の「関連用語」チップをクリック
2. 現在のカードがフェードアウト
3. 関連用語のカードがフェードイン
4. 「戻る」ボタンで元のカードに戻れる

【履歴管理】
- ジャンプ履歴をスタックで管理
- 複数回ジャンプしても順番に戻れる
- パンくずリスト形式で履歴を表示
```

## 8. Phase 6：仕上げとデプロイ

### 8.1 レスポンシブ対応

**Bolt.newへの指示：**

```
スマートフォンでも快適に使えるよう、レスポンシブデザインを適用してください。

【ブレークポイント】
- スマホ：〜640px
- タブレット：641px〜1024px
- PC：1025px〜

【スマホ向けの調整】
- カードは画面幅いっぱいに表示
- ナビゲーションは下部固定のタブバー形式
- フォントサイズを若干大きめに
- タップ領域は44px以上確保
```

### 8.2 パフォーマンス最適化

**Bolt.newへの指示：**

```
アプリのパフォーマンスを最適化してください。

【実施項目】
- 画像の遅延読み込み（Lazy Loading）
- 用語データの分割読み込み（章ごと）
- アニメーションのGPUアクセラレーション
- 不要な再レンダリングの防止（useMemo, useCallback）
```

### 8.3 デプロイ

**Bolt.newからのデプロイ手順：**

1. **Bolt.new内でデプロイ**
   - 右上の「Deploy」ボタンをクリック
   - Netlifyが自動で設定される
   - URLが発行される（例：your-app.netlify.app）

2. **カスタムドメイン設定（任意）**
   - Netlifyダッシュボードでドメイン設定
   - DNSレコードを追加

3. **公開前チェックリスト**
   - [ ] 全章のデータが正しく表示される
   - [ ] 進捗が正しく保存される
   - [ ] スマホで問題なく動作する
   - [ ] 読み込み速度が許容範囲内

## 10. トラブルシューティング

### よくある問題と解決策

| 問題 | 原因 | 解決策 |
|------|------|--------|
| 画面が真っ白 | JavaScriptエラー | コンソールを確認し、エラーを報告 |
| データが保存されない | LocalStorage制限 | データ量を確認、古いデータを削除 |
| アニメーションがカクカク | パフォーマンス不足 | will-changeプロパティを追加 |
| スマホで表示崩れ | CSS未対応 | メディアクエリを追加依頼 |

### Bolt.newが期待通りに動かない時

1. **指示を分割する**
   - 大きな機能は小さなステップに分けて依頼

2. **具体例を示す**
   - 「こんな感じで」と参考画像やコードを添付

3. **現状を正確に伝える**
   - 何が動いていて、何が動いていないか明確に

4. **リセットも選択肢**
   - 行き詰まったら、その部分だけ作り直しを依頼
