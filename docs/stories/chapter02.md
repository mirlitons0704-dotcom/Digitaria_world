# 第2章：ファイル王国の図書館

**〜 拡張子が教えてくれること 〜**

**カテゴリ：ファイル形式（File Types & Formats）**
**登場用語：.TXT, .MD, .HTML, .CSS, .JS, .TS, .JSON, .JPG/JPEG, .PNG, .SVG, .PDF, .CSV, .YAML, .env, .SCSS, .TSX, .JSX, .SQL, Format, Metadata, File metadata, Config file, Configuration, Tech stack, README**

---

## シーン1：図書館の門

```
ビットくんに連れられて、巨大な図書館の前に来た。
門には「ファイル王国立図書館」と書かれている。

「ここには、デジタリアのあらゆるファイルが
収められているんだ」

中に入ると、司書のファイラが迎えてくれた。

「ようこそ、旅の方。
この図書館では、ファイルを『拡張子』ごとに
分類して保管しています」

「拡張子って、ファイル名の最後についてる
ドット以降のこと？」

「その通りです。
拡張子を見れば、そのファイルが
どんな種類のデータかわかるんですよ」
```

---

## シーン2：文字だけの部屋

```
最初の部屋には、シンプルな書類が並んでいた。

「こちらは『テキストファイル』の部屋です。
.TXT は最もシンプルなファイル形式で、
装飾のない文字だけを保存します」

「メモ帳で作るファイルだ！」

「はい。どんなコンピュータでも開けるのが特徴です。
ただし、太字にしたり色をつけたりはできません」

隣の棚を指さすと、少し変わったファイルがあった。

「こちらは .MD——Markdown（マークダウン）形式です。
テキストファイルなのに、簡単な記号で
見出しや箇条書きを表現できるんですよ」

# こんな風に
- 見出しや
- リストが
- 作れます

「プログラマーに人気のある形式です」
```

**💡 用語登場：.TXT, .MD**

---

## シーン3：Webの部屋

```
次の部屋は、カラフルで華やかだった。

「ここは『Web系ファイル』の部屋です」

ファイラが3つのファイルを見せてくれた。

「.HTML は Web ページの『骨組み』を作るファイル。
家で言えば、柱や壁の位置を決める設計図です」

「.CSS は『装飾』を担当します。
壁紙の色、家具の配置、カーテンの柄——
見た目を美しく整えます」

「そして .JS（JavaScript）は『動き』を加えます。
ドアを開けたら電気がつく、
ボタンを押したら音楽が流れる……
そんな仕掛けを作るんです」

「3つで役割分担しているんだね！」

「素晴らしい理解です。
そして .TS（TypeScript）は、
JavaScript をより安全に書くための
改良版だと思ってください」
```

**💡 用語登場：.HTML, .CSS, .JS, .TS**

---

## シーン4：データの部屋

```
次の部屋には、整然と並んだ箱があった。

「こちらは『データ形式』の部屋です。
.JSON（ジェイソン）は、
データを整理して保存する人気の形式です」

ファイラが中身を見せてくれた。

{
  "name": "ビットくん",
  "age": 1,
  "friends": ["バイトさん", "ファイラ"]
}

「人間にも読めるし、
コンピュータにも理解しやすい。
両方に優しい形式なんです」

「.CSV は、表のデータを保存する形式です。
Excel で開けるので、
ビジネスでよく使われます」

名前,年齢,役割
ビットくん,1,ガイド
バイトさん,8,まとめ役

「カンマで区切って並べるだけなので、
とてもシンプルです」
```

**💡 用語登場：.JSON, .CSV**

---

## シーン5：画像の部屋

```
「ここは『画像ファイル』の部屋です」

壁にはたくさんの絵が飾られていた。

「.JPG（ジェイペグ）は写真向けの形式です。
ファイルサイズを小さくできますが、
保存するたびに少しずつ画質が落ちます」

「.PNG（ピング）は、透明な部分を保存できます。
ロゴやアイコンによく使われますね。
画質も落ちません」

「.SVG は特別です。
どれだけ拡大しても、絶対にぼやけません。
数学の式で図形を表現しているからです」

「拡大してもぼやけないって、魔法みたい！」

「アイコンやロゴには最適ですよ」
```

**💡 用語登場：.JPG/JPEG, .PNG, .SVG**

---

## シーン6：特別な書類の部屋

```
重厚な扉の向こうは、ビジネス書類の部屋だった。

「.PDF は、どんなコンピュータで開いても
同じ見た目で表示される形式です。
契約書や報告書に使われます」

「編集しにくいのも特徴ですね。
『完成した書類』を渡すときに便利です」

ファイラが小さなファイルを見せてくれた。

「こちらは設定ファイルたちです」

「.YAML（ヤムル）は、
設定を人間が読みやすく書く形式です」

「.env（エンブ）は、
パスワードなど秘密の情報を
保存するためのファイルです。
絶対に他人に見せてはいけません」

「秘密のファイルがあるんだ……」
```

**💡 用語登場：.PDF, .YAML, .env**

---

## シーン7：Reactの部屋

```
「こちらは少し上級者向けの部屋です」

ファイラが説明を続けた。

「.JSX と .TSX は、
React というツールで使う特別な形式です。
HTML のような見た目のコードを
JavaScript の中に書けるんです」

「.SCSS は、CSS をもっと便利に書くための形式です。
変数を使ったり、入れ子にしたり……
大きなプロジェクトで重宝します」

「.SQL（エスキューエル）は、
データベースに命令を送るための言語です。
『この条件のデータをちょうだい』と
お願いするときに使います」

「いろんな種類があるんだなあ……」
```

**💡 用語登場：.JSX, .TSX, .SCSS, .SQL**

---

## シーン8：ファイルの情報

```
「さて、ファイルには『見えない情報』も
含まれているんですよ」

ファイラが不思議な虫眼鏡を渡してくれた。
ファイルを覗くと、隠れた文字が見えた。

「これは Metadata（メタデータ）——
『データについてのデータ』です」

「作成日時、更新日時、ファイルサイズ、
作者の名前……
本で言えば、奥付のような情報ですね」

「普段は見えないけど、
裏側にはいろんな情報が記録されているんだ！」
```

**💡 用語登場：Metadata, File metadata**

---

## シーン9：設定ファイルたち

```
「プロジェクトを管理するには、
Config file（設定ファイル）が欠かせません」

「Configuration（設定・構成）を
ひとつのファイルにまとめておくと、
チーム全員が同じ環境で作業できます」

「そして README（リードミー）——
プロジェクトの説明書です。
『最初にこれを読んでね』という
大切なファイルなんです」

「新しいプロジェクトを見るときは、
まず README を読む。
これが基本です」
```

**💡 用語登場：Config file, Configuration, README**

---

## シーン10：技術の組み合わせ

```
図書館を出る前に、ファイラが最後の説明をしてくれた。

「アプリを作るときに使う
技術の組み合わせを Tech stack（技術スタック）
と呼びます」

「たとえば、
React + TypeScript + Tailwind CSS
という組み合わせは、
とても人気のある技術スタックです」

「データの Format（形式）を理解して、
適切な技術スタックを選ぶ。
これができれば、
いろんなアプリが作れるようになりますよ」

「ありがとう、ファイラさん！」

「また来てくださいね。
次は、データの『型』について
学ぶ旅が待っていますよ」
```

**💡 用語登場：Tech stack, Format**
