## シーン3：エラーと向き合う勇気

```
最初の蝶が、そっと光を放った。

    ✧ Try/catch ✧

「プログラムは、時に失敗する」

「でも、Try/catch（トライキャッチ）があれば
大丈夫」

try {
  // 失敗するかもしれないコード
  riskyOperation();
} catch (error) {
  // 失敗したときの対処
  console.log('エラーが起きたよ:', error);
}

「tryで挑戦して、
catchで失敗を受け止める」

「失敗を恐れないで。
ちゃんと受け止める準備があれば、
挑戦できるから」

あなたは頷いた。

次の蝶が舞い降りる。

    ✧ Null / Undefined ✧

「『ない』にも種類があるんだよ」

「null（ヌル）は、
『意図的に空っぽ』」

「undefined（アンディファインド）は、
『まだ何も入っていない』」

let box = null;      // 空の箱を用意した
let something;       // 箱すらまだない

「似ているけど、意味が違う。
この違いを知っておくと、
バグの原因が見つけやすくなるよ」
```

## シーン4：モダンな書き方

```
3匹目の蝶が、優しく光る。

    ✧ Optional chaining (?.) ✧

「深いオブジェクトにアクセスするとき、
途中がnullだとエラーになる」

「でも、?.（オプショナルチェイニング）を
使えば安全だよ」

// 危険なアクセス
user.address.city  // addressがないとエラー！

// 安全なアクセス
user?.address?.city  // undefinedが返るだけ

「『いますか？』と確認しながら進む、
慎重なノックのようなものだね」


4匹目の蝶が続く。

    ✧ Nullish coalescing (??) ✧

「値がnullやundefinedのとき、
代わりの値を使いたい」

const name = user.name ?? 'ゲスト';
// nameがnull/undefinedなら'ゲスト'を使う

「||と似てるけど、
??は0や空文字を『ある』と判断するんだ」

const count = 0;
count || 10   // → 10（0はfalsyだから）
count ?? 10   // → 0（0は『ある』から）

「この違い、大切だよ」


5匹目。

    ✧ Template literal ✧

「文字列に変数を埋め込みたいとき、
バッククォート（`）を使うんだ」

const name = '太郎';
const message = `こんにちは、${name}さん！`;
// → "こんにちは、太郎さん！"

「+で繋げるより、ずっと読みやすいでしょ？」

「複数行もそのまま書けるんだよ」

const poem = `
  コードを書く
  蝶が生まれる
  知識が育つ
`;


6匹目。

    ✧ Ternary operator ✧

「条件で値を変えたいとき、
三項演算子が便利」

const status = isLoggedIn ? 'ログイン中' : 'ゲスト';

「条件 ? 真のとき : 偽のとき」

「if-elseの短縮形だね。
Reactでもよく使うよ」

{isLoggedIn ? <Dashboard /> : <Login />}
```

## シーン5：ブラウザの世界

```
7匹目の蝶が、温かく光る。

    ✧ localStorage ✧

「ブラウザにデータを保存したい？
localStorageが使えるよ」

// 保存
localStorage.setItem('theme', 'dark');

// 取得
const theme = localStorage.getItem('theme');

// 削除
localStorage.removeItem('theme');

「ブラウザを閉じても消えないんだ。
テーマ設定やお気に入りの保存に便利」

「ただし、文字列しか保存できないから、
オブジェクトはJSONにしてね」

localStorage.setItem(
  'user',
  JSON.stringify({ name: '太郎', level: 5 })
);
```

## シーン6：レイアウトの魔法

```
8匹目、9匹目、10匹目、11匹目——
4匹の蝶が連なって舞い降りた。

「CSSのレイアウト、教えるね」


    ✧ Flexbox ✧

「要素を横に並べたい？
Flexboxを使おう」

.container {
  display: flex;
  justify-content: center;  /* 横方向の配置 */
  align-items: center;      /* 縦方向の配置 */
  gap: 16px;                /* 要素間の隙間 */
}

「ダンサーを整列させるように、
要素を自由自在に並べられるんだ」


    ✧ Grid (CSS) ✧

「2次元のレイアウトには
Gridが強力だよ」

.container {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr; /* 3列 */
  gap: 20px;
}

「エクセルのセルみたいに、
縦横に要素を配置できる」


    ✧ z-index ✧

「要素が重なったとき、
どっちが前に来る？」

「z-indexで決めるんだ」

.modal {
  z-index: 100;  /* 大きいほど前面 */
}

.overlay {
  z-index: 99;   /* modalより後ろ */
}

「数字が大きいほど、手前に来るよ」


    ✧ Box model ✧

「全ての要素は、4つの層でできている」

┌─────── margin ────────┐
│ ┌──── border ─────┐ │
│ │ ┌── padding ──┐ │ │
│ │ │  content   │ │ │
│ │ └─────────────┘ │ │
│ └─────────────────┘ │
└───────────────────────┘

「content（中身）、
padding（内側の余白）、
border（枠線）、
margin（外側の余白）」

「これを理解すると、
レイアウトが思い通りになるよ」
```

## シーン7：画面サイズへの適応

```
12匹目と13匹目が、一緒に舞い降りた。

    ✧ Responsive design ✧

「スマホでもPCでも
きれいに見せたい」

「それがレスポンシブデザイン」

「画面サイズに応じて、
レイアウトを変えるんだ」


    ✧ Breakpoint (CSS) ✧

「レイアウトが切り替わる境界を
ブレークポイントと呼ぶよ」

/* スマホ（デフォルト） */
.container {
  flex-direction: column;
}

/* タブレット以上（768px〜） */
@media (min-width: 768px) {
  .container {
    flex-direction: row;
  }
}

「Tailwindなら、md:やlg:で
簡単に指定できるよ」

<div className="flex-col md:flex-row">

「小さい画面から作り始める、
『モバイルファースト』がおすすめ」
```
